Version corrigée

Cette version comporte des indications pour les réponses aux exercices.

PreReq

Avoir fait le TP de hiérarchie de classes Compte. Maitriser l’héritage.

ObjTP

Générer la documentation de classes pour les clients de la classe (API).

Durée

1 séance de 1,5h sur machine.

À rendre

A déposer : code source de CompteDepot et CompteSecurise commentées et, dans un répertoire, la documentation générée par javadoc pour toutes les classes (un seul fichier .zip).

Pour les profs :

  • But du TP : Documenter des classes, ici 2 classes de la hiérarchie de comptes.

Faire avec eux le début :

  1. Explications des principes

  2. Commande javadoc

  3. Récupérer les fichiers, lancer la commande javadoc du style : 

    javadoc  -version  -author  -d  documentation   *.java

  4. Commentaires javadoc (format)

Insister sur les principes de documentation :

  • Doc avant l'élément documenté

  • Tags possible

  • Principe de la doc :

    • A destination : d’un programmeur

    • A destination d’un client qui n’a pas à connaître l’implémentation mais "l’usage" de la classe.

Déroulement prévu :

  • 10' : rapatrier tous les fichiers nécessaires

  • 10' : principes de documentation des classes

  • 20' : premier test javadoc

  • 45' : documentation des classes CompteDepot/CompteSecurise, au moins CompteDepot

But du TP : Documenter des classes, ici quelques classes de la hiérarchie de comptes.

A Suivre de près dans le TP :

Le contenu de la documentation produite : pas l’implémentation MAIS l’utilisation pour un client de la classe. Faire au moins CompteDepot.
