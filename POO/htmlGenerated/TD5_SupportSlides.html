1. Principes

Une structure autorferrentielle (parfois appele structure rcursive) correspond une structure dont au moins un des champs contient une rfrence vers une structure de mme type.

De cette faon on cre des lments cellule (appels parfois noeuds ou liens) contenant des donnes, mais, contrairement un tableau, celles-ci peuvent tre parpilles en mmoire et relies entre elles par des liens logiques (des rfrences), c’est--dire un ou plusieurs champs dans chaque structure contenant la rfrence d’une ou plusieurs structures de mme type.

Plusieurs cas sont possibles :

  • Lorsque la structure contient des donnes et une rfrence vers la structure suivante : on parle de structure/liste chane.
  • Lorsque la structure contient des donnes, une rfrence vers la structure suivante, et une rfrence vers la structure prcdente : on parle de structure/liste chane double.
  • Lorsque la structure contient des donnes, une rfrence vers une premire structure suivante, et une rfrence vers une seconde, on parle d’arbre binaire.

public class Cellule {
        private double contenu;
        private Cellule suivant;

        public Cellule (double pVal, Cellule pSuiv) {
                this.contenu = pVal;
                this.suivant = pSuiv;
        }
        public double getContenu () {
                return this.contenu;
        }
        public Cellule getSuivant () {
                return this.suivant;
        }
        public void setContenu (double pVal) {
                this.contenu = pVal;
        }
        public void setSuivant (Cellule pSuiv) {
                this.suivant = pSuiv;
        }
}

4. Diffrence tableau "dynamique" <> liste chane simple

Donner les diffrences entre tableaux "dynamiques" et listes chanes simples dans les situations suivantes :

  1. Ajout d’un lment en fin.
  2. Ajout au dbut.
  3. Suppression d’un lment (dbut, milieu, fin).
  4. Accs lment i.
Rponses
  1. Ajout d’un lment en fin.
    • Tableau : si plein ⇒ crer un autre tableau + recopie, sinon une instruction.
    • Liste chane : parcours liste + ajout une cellule. On pourrait aussi conserver la rfrence de la fin de la liste.
  2. Ajout au dbut.
    • Tableau : si plein ⇒ crer un autre tableau + recopie/dcalage, si non plein ⇒ dcalage.
    • Liste chane : immdiat ⇒ cration d’une cellule.
  3. Suppression d’un lment (dbut, milieu, fin).
    • Tableau : dcalage dans le tableau (sauf fin) + ventuellement diminution de la taille du tableau.
    • Liste chane : rechercher (parcours) le prcdent et modification d’une rfrence
  4. Accs lment i.
    • Tableau : immdiat
    • Liste chane : parcours des (i-1) lments prcdents.
