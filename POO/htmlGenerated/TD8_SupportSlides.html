2. Constructeurs

Question : Écrire les constructeurs des classes Voiture et Camion. On écrira un "minimum" de code.

Les valeurs par défaut seront "--" pour les chaînes de caractères, 0 pour les numériques.

Solution
class Voiture extends Vehicule   {
   private double tauxUsure;
   private String marque;

   public Voiture ()   {
       this ("---- -- --", 0);
   }

   public Voiture (String pImmat, int pPuiss)   {
       this (pImmat, pPuiss, "--", 0);
   }
   public Voiture (String pImmat, int pPuiss, String pMarque,
                                double pTauxUsure)    {
       super (pImmat, pPuiss); // Appel constructeur superclasse
       this.marque = pMarque;
       this.tauxUsure = pTauxUsure;
   }
        // ...
}

class Camion extends Vehicule  {
   private int chargeUtile;
   private double ratio ;

   public Camion () {
       this ("---- -- --", 0);
   }

   public Camion (String pImmat, int pPuiss) {
       this (pImmat, pPuiss, 1000, 1.0);
   }

   public Camion (String pImmat, int pPuiss, int pChargeU,
                        double pRatio) {
       super (pImmat, pPuiss); // Appel constructeur superclasse
       this.chargeUtile = pChargeU;
       this.ratio = pRatio;
   }
        //...
}

3. Méthodes coutTotal() et coutSuppl()

Les spécialistes donnent les spécification suivantes :

  1. Classe Vehicule
    1. Le coût total d’un Vehicule est de 15% de la distance parcourue fois la consommation.
  2. Classe Voiture
    1. Le coût total d’une Voiture est l’addition de :
      • coût total d’un véhicule,
      • un forfait de coût de 0.85 fois la distance parcourue,
      • le coût supplémentaire particulier aux voitures.
    2. Le coût supplémentaire particulier aux voitures est la distance parcourue multipliée par le taux d’usure (en pourcentage).
  3. Classe Camion
    1. Le coût total d’un Camion est l’addition de :
      • coût total d’un véhicule,
      • un forfait de 1.35 fois la distance parcourue,
      • le coût supplémentaire particulier aux camions.
    2. Le coût supplémentaire particulier aux camions est la distance parcourue multipliée par le ratio (en pourcentage) et la charge utile.

On souhaite faire fonctionner le code suivant :

Vehicule tabv[] = new Vehicule [4];

tabv[0] = new Voiture ("2222 VV 22", 5);
tabv[1] = new Camion ("2222 CC 22", 12);
tabv[2] = new Voiture ("3333 VV 33", 6, "Opel", 10);
tabv[3] = new Camion ("3333 CC 33", 14, 1000, 2);

for (int i=0 ; i<4 ; i++) {
        tabv[i].afficher();
        System.out.println (tabv[i].coutTotal());
        System.out.println (tabv[i].coutSuppl());
}

Questions :

  1. Écrire les méthodes coutTotal() et coutSuppl() dans les 3 classes.
  2. Nous n’avons pas de définition pour le coût supplémentaire particulier aux véhicules. Il n’existe pas. Doit-on déclarer la méthode coutSuppl() dans la classe Vehicule ? Comment l'écrire ?
Solution

Écrire coutSuppl() dans la classe Vehicule :

  1. Nécessaire pour que le code de test marche : System.out.println (tabv[i].coutSuppl());
  2. On ne peut "que" mettre 0. 
class Vehicule
{
        // ...
   public double coutTotal ()  {
       return this.distParcourue * this.consommation * 0.15;
   }
   public double coutSuppl () {
       return 0;
   }
}

class Voiture extends Vehicule
{
        // ...
        public double coutTotal ()  {
                return super.coutTotal() + this.coutSuppl()
                                + this.getKilometrage() * 0.85;
        }
        public double coutSuppl ()  {
                return this.getKilometrage() * this.tauxUsure/100 ;
        }
} // Fin Classe Voiture

class Camion extends Vehicule
{
        // ...

        public double coutTotal ()      {
                return super.coutTotal() + this.coutSuppl()
                        + this.getKilometrage() * 1.35 ;
        }
        public double coutSuppl ()      {
                return this.getKilometrage () * this.ratio / 100 ;
        }
} // Fin Classe Camion

4. Classes abstraites

On peut constater plusieurs éléments :

  1. Point de vue client (garagiste, …) : un objet Vehicule n’existe pas ⇒ seules existent vraiment une voiture ou un camion.
  2. Point de vue client (garagiste, …) : on parle des véhicules en général pour signifier un ensemble de voitures et camions (le parc du garage, …).
  3. La classe Vehicule :
    1. Point de vue fonctionnel : généralise des attributs et méthodes communs à tous les véhicules (voitures et camions).
    2. Point de vue ensembliste : permet de désigner "indifféremment" des voitures et des camions.
Solution

Cours en ligne → Abstract diapos 35-42

Question :

  1. Déclarer la classe Vehicule abstraite.
  2. Modifier la déclaration de coutSuppl() dans Vehicule en conséquence.
Solution

public abstract class Vehicule

{

public double coutTotal () { return this.distParcourue * this.consommation * 0.15; }

public abstract double coutSuppl () ; { return 0; }

}
