Code source à déposer : Classes produites (un seul fichier zip) + réponses au questions éventuelles en commentaire en début de fichier.

1. Objectif

Le but de ce TP est de construire une hiérarchie simple de classes avec héritage et de tester les classes produites.

On souhaite construire l’arborescence suivante :

FormeGraphique
    Cercle
        Cylindre
        Sphere
    Rectangle
        Carré

2. Travail à faire

2.1. Code à écrire

Pour chaque classe, une à la fois, dans l’ordre proposé (Sphere et Carre tout à la fin) :

  1. Écrire la classe selon les recommandations données ci-après

    On ne traitera pas de cas d’erreurs avec levée d’exceptions dans ce sujet.

  2. Écrire un petit test de la classe dans une méthode main()

    • Créer un objet avec chaque constructeur.

    • Pour chaque objet créé :

      • Lui appliquer la méthode afficher().

      • Lui appliquer les méthodes transformateurs de la classe.

      • Lui appliquer les méthodes accesseurs de la classe (dans un System.out.println()).

      • Lui appliquer les méthodes transformateurs héritées.

      • Lui appliquer les méthodes accesseurs héritées.

      • Lui appliquer la méthode afficher().

  3. Vérifier les valeurs affichées à l'écran ! Elles doivent être correctes.

    Au besoin : commenter au fur et à mesure le code écrit pour tester chaque classe dans le main().

    MAIS NE PAS LES EFFACER, SERA UTILE A LA FIN !!

2.2. Spécification (simple) de chaque classe

2.2.1. La classe FormeGraphique

La classe FormeGraphique définit une forme graphique par ses coordonnées sur le plan x et y (entiers).

Écrire la classe FormeGraphique. On définira :

  1. le(s) attribut(s) nécessaire(s),

  2. un constructeur par défaut initialisant les coordonnées à 0,

  3. un constructeur paramétré permettant d’initialiser correctement les coordonnées de la FormeGraphique,

  4. une méthode accesseur (get) pour chaque attribut défini,

  5. une méthode transformateur (set) pour chaque attribut défini,

  6. la méthode qui calcule la surface d’une forme, valeur de type double. A ce niveau, elle est de 0,

  7. la méthode qui calcule le volume d’une forme, valeur de type double. A ce niveau, il est de 0,

  8. la méthode afficher permettant d’afficher à l'écran le texte suivant :

    Forme graphique : abscisse * valeur abcisse *ordonnée valeur ordonnée

    Exemple :

    Forme graphique : abscisse 245 ordonnée 189

2.2.2. La classe Cercle

La classe Cercle est une forme graphique avec en plus un rayon (entier).

Écrire la classe Cercle. On définira :

  1. la classe Cercle comme héritant de FormeGraphique,

  2. le(s) attribut(s) nécessaire(s),

  3. un constructeur par défaut initialisant les coordonnées à 0 ainsi que le rayon à 0,

    • voir le cours pour les appels des constructeurs de la super-classe,

  4. un constructeur paramétré permettant d’initialiser correctement toutes les données d’un Cercle,

  5. une méthode accesseur (get) pour chaque attribut défini,

  6. une méthode transformateur (set) pour chaque attribut défini,

  7. la méthode qui calcule la surface d’un Cercle (rappel : PI x rayon2),

    • utiliser la constante PI de la classe Math : Math.PI,

  8. la méthode afficher permettant d’afficher à l'écran le texte suivant. On appelera obligatoirement la méthode afficher() de la classe parente.

    Forme graphique : abscisse valeur abcisse ordonnée valeur ordonnée

    Cercle : rayon valeur rayon

    Exemple :

    Forme graphique : abscisse 245 ordonnée 189
Cercle : rayon 17

2.2.3. La classe Cylindre

La classe Cylindre est définie comme un cercle avec en plus une hauteur (entier).

Écrire la classe Cylindre. On définira :

  1. un héritage éventuel pour la classe (assez évident),

  2. le(s) attribut(s) nécessaire(s),

  3. un constructeur par défaut initialisant les coordonnées à 0, le rayon à 0 ainsi que la hauteur à 0,

  4. un constructeur paramétré permettant d’initialiser correctement toutes les données d’un Cylindre,

  5. une méthode accesseur (get) pour chaque attribut défini,

  6. une méthode transformateur (set) pour chaque attribut défini,

  7. la méthode qui calcule la surface d’un Cylindre (rappel : 2 x PI x rayon x hauteur + 2 x PI x rayon2),

  8. la méthode qui calcule le volume d’un Cylindre (rappel : PI x rayon2 x hauteur),

  9. la méthode afficher permettant d’afficher à l'écran le texte suivant. On appelera obligatoirement la méthode afficher() de la classe parente.

    Forme graphique : abscisse valeur abcisse ordonnée valeur ordonnée

    Cercle : rayon valeur rayon

    Cylindre : hauteur ~valeur hauteur

    Exemple :

    Forme graphique : abscisse 245 ordonnée 189
Cercle : rayon 17
Cylindre : hauteur 789

2.2.4. La classe Rectangle

La classe Rectangle est une FormeGraphique comprenant des coordonnées sur le plan x et y (entiers), une hauteur et une largeur (entiers).

Écrire la classe Rectangle. On définira :

  1. un héritage éventuel pour la classe,

  2. le(s) attribut(s) nécessaire(s),

  3. un constructeur par défaut initialisant les coordonnées à 0 ainsi que hauteur/largeur à 0,

  4. un constructeur paramétré permettant d’initialiser correctement toutes les données d’un Rectangle,

  5. une méthode accesseur (get) pour chaque attribut défini,

  6. une méthode transformateur (set) pour chaque attribut défini,

  7. la méthode qui calcule la surface d’un Rectangle (rappel : hauteur x largeur),

  8. la méthode afficher permettant d’afficher à l'écran le texte suivant. On appelera obligatoirement la méthode afficher() de la classe parente.

    Forme graphique : abscisse valeur abcisse ordonnée valeur ordonnée

    Rectangle : hauteur valeur hauteur largeur valeur largeur

    Exemple :

    Forme graphique : abscisse 245 ordonnée 189
Rectangle : hauteur 43 largeur 128

2.2.5. La classe Sphere

La classe Sphere est définie comme un cercle et a une surface et un volume.

Écrire la classe Cercle. On définira :

  1. un héritage éventuel pour la classe,

  2. le(s) attribut(s) nécessaire(s), réfléchir …

  3. un constructeur par défaut initialisant les coordonnées à 0 ainsi que le rayon à 0,

  4. un constructeur paramétré permettant d’initialiser correctement toutes les données d’une Sphere,

  5. une méthode accesseur (get) pour chaque attribut défini,

  6. une méthode transformateur (set) pour chaque attribut défini,

  7. la méthode qui calcule la surface d’une Sphere (rappel : 4 x PI x rayon2),

  8. la méthode qui calcule le volume d’une Sphere (rappel : 4 x PI x rayon3 / 3),

  9. la méthode afficher permettant d’afficher à l'écran le texte suivant. On appelera obligatoirement la méthode afficher() de la classe parente.

    Forme graphique : abscisse valeur abcisse ordonnée valeur ordonnée

    Cercle : rayon valeur rayon

    Sphere : rayon ~valeur hauteur

    Exemple :

    Forme graphique : abscisse 245 ordonnée 189
Cercle : rayon 17
Sphere : rayon 17

2.2.6. La classe Carre

La classe Carre est un rectangle particulier puisque la longueur des 2 cotés sont égales.

Écrire la classe Carre. On définira :

  1. un héritage éventuel pour la classe,

  2. le(s) attribut(s) nécessaire(s),

  3. un constructeur par défaut initialisant les coordonnées à 0 ainsi que le côté à 0,

  4. un constructeur paramétré permettant d’initialiser correctement un carré : les coordonnées et la longueur du côté,

  5. une méthode accesseur (get) pour chaque attribut défini,

  6. une méthode transformateur (set) pour chaque attribut défini,

  7. à réfléchir : la méthode qui calcule la surface d’un carré (rappel : coté x coté),

  8. la méthode afficher permettant d’afficher à l'écran le texte suivant. On appelera obligatoirement la méthode afficher() de la classe parente.

    Forme graphique : abscisse valeur abcisse ordonnée valeur ordonnée

    Rectangle : hauteur valeur hauteur largeur valeur largeur

    Carré : côté valeur coté Exemple :

    Forme graphique : abscisse 245 ordonnée 189
Rectangle : hauteur 43 largeur 43
Carré : côté 43

Compte tenu de l’héritage utilisé, en utilisant les méthodes transformateurs héritées : arrive-t-on à faire des "carrés qui ne soient pas vraiment carrés" ?

3. Pour en arriver au polymorphisme

A la fin du main() écrit :

  1. Vous devriez avoir créé 12 objets (12 new, 2 par classe).

  2. Insérer la déclaration suivante pour créer un tableau de 12 éléments et y ranger les formes graphiques créées : 

    FormeGraphique tabFormes[];

tabFormes = new FormeGraphique [12];

tabFormes[0] = maPremiereFormeGraphique; // variable référençant la première FormeGraphique créée
tabFormes[1] = maDeuxiemeFormeGraphique; // variable référençant la deuxième FormeGraphique créée
tabFormes[3] = monPremierCercle; // variable référençant le premier Cercle créé
tabFormes[4] = monDeuxiemeCercle; // variable référençant le deuxième Cercle créé
tabFormes[5] = monPremierCylindre; // variable référençant le premier Cylindre créé
... // continuer

  3. Continuer le programme précédent pour y mettre tous les objets créés.

  4. Compiler. ok ?

  5. Faire une boucle pour appliquer surface(), volume() et afficher() à chaque élément du tableau. Quel est le résultat ?

4. Utiliser les méthodes de la super-classe

Dans la classe Cylindre :

  1. Mettre en commentaire le corps de la méthode surface() et volume()

  2. En écrire une nouvelle version en utilisant la méthode surface() définie dans la classe Cercle. Bien regarder les formules.

On pourra faire de même dans la classe Sphere.
