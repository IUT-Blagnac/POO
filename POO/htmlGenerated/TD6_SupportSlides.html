En TP, nous avons dj travaill le cas de l’association oriente (navigable dans un seul sens) ou agrgation dans le cas de la classe AgenceBancaire :

Pour mettre en oeuvre une association, le principe est toujours de :

  • pour la navigabilit : stocker les "liens" d’une classe vers une autre par un attribut, et ce dans les deux classes,
  • selon la multiplicit : l’attribut se voit changer de type (objet, tableau/ArrayList, …),
  • il faudra crer des mthodes permettant de maintenir les relations en "tat cohrent" par rapport aux multiplicits.

2.2. Une version cohrente : maintient des liens

Pour rsoudre le problme et maintenir les liens cohrents : il faut que lors de l’association d’une Commande un Client :

  • associer la Commande affecte au Client,
  • associer le Client la Commande,
  • si le Client est dj associ une Commande, l’association de cette Commande vers ce Client doit tre rompue,
  • si la Commande affecte est dj associe un Client, cette association doit tre rompue.

Il en est de mme lorsqu’un Client est associ une Commande.

Question : Pour raliser ce traitement, crire :

  • dans la classe Commande une nouvelle mthode void definirClient (Client cli),
  • dans la classe Client, une nouvelle mthode duale de fonctionnement similaire : public void definirCommande (Commande comm).

La mthode void definirClient (Client cli) dans la classe Commande :

  • Vrifie si le paramtre cli est null : auquel cas ne rien faire.
  • Si le Client cli est associ une Commande : dfaire cette association (associer cette Commande au Client null (setClient(null))).
  • Si la Commande est dj associe un Client : dfaire cette association (associer ce Client la Commande null (setCommande(null))).
  • Associer la Commande et le Client cli.

public void definirClient (Client cli) {
        if( cli != null ){
                if (cli.getCommande() != null ) {
                        cli.getCommande().setClient(null);

                        this.leClient.setCommande(null);

  • dans la classe Client, une nouvelle mthode duale : void ajouterUneCommande (Commande comm).

Ces deux mthodes doivent permettre de dfaire et construire correctement les liens entre objets afin de maintenir les associations cohrentes (viter le problme du cas prcdent)

Rponse

La mthode void definirClient (Client cli) dans la classe Commande :

  • Vrifie si le paramtre cli est null : auquel cas ne rien faire.
  • Si la Commande est dj associe un Client : dfaire cette association (enlever la Commande au Client (removeCommande())).
  • Associer la Commande et le Client cli.

La mthode public void ajouterUneCommande (Commande comm) dans la classe Client :

  • Vrifie si le paramtre comm est null : auquel cas ne rien faire.
  • Vrifie que la Commande comm ne soit pas dj enregistre comme Commande de ce Client : auquel cas ne rien faire (existeCommande()).
  • Si la Commande comm est associ un Client : dfaire cette association (removeCommande()).
  • Associer la Commande et le Client cli. 
class Commande {
        // ...
        public void definirClient (Client cli) {
            if( cli != null ){
                if (! cli.existeCommande(this)) {

                                this.leClient.removeCommande(this);
                        }
                        this.leClient = cli; // Ou this.setClient(cli);
                        cli.addCommande(this);
                }
            }
        }
}
