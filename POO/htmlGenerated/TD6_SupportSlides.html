1. Domaine d'tude

On s’intresse dans ce TD la mise en oeuvre d’une classe Ensemble ordonn de chanes de caractres.

Un ensemble ordonn est une collection ordonne qui n’accepte pas les doublons.

  • Par exemple, un ensemble de String n’accepte qu’une seule fois la valeur "bonjour". Par contre "bonjour" et "BONjour" sont acceptes.
  • L’ensemble constitu des trois valeurs "a", "d", "b", "c" sera ordonn en { "a", "b", "c", "d" }.
  • On supposera dans tout l’exercice que la valeur null est refuse (non ajoute, non enlev, non recherche).

Question : Ecrire les mthodes de la classe

  • On vous donne la mthode trouveRang(String s)
  • Ecrire dans l’ordre : le constructeur, size, contains, add, toArray. remove a dj t crite en TP (Agence bancaire).
  • Ecrire les mthodes qui renvoient un nouvel objet OrderedStringSet : union (receveur union paramtre), difference (receveur "moins" paramtre), intersect (receveur intersection paramtre)
    • Ces mthodes sont elles des transformateurs ? des accesseurs (observateurs) ?
Rponse

La mthode void associerClient (Client cli) dans la classe Commande ralise le traitement :

  • Si le paramtre cli est null : dfaire l’association :
    • Si la Commande est dj associe un Client : enlever la Commande de ce Client.
    • Lier la Commande null (setClient()).
  • Si le paramtre cli n’est pas null : construire une association :
    • Si la Commande est dj associe un Client : enlever la Commande de ce Client.
    • Crer les liens pour associer la Commande et cli (setClient() et addCommande()). 
class Commande {
        // ...
        public void associerClient (Client cli) {
            if (cli == null ){ // Dfaire l'association
                if (this.leClient != null) {
                        this.leClient.removeCommande(this);
                }
                this.setClient(null); // Ou this.leClient = null ;
            } else { // Crer l'association
                if (this.leClient != null) {
                        this.leClient.removeCommande(this);
                }
                this.setClient(cli); // Ou this.leClient = cli;
                cli.addCommande(this);
            }
        }
}

//A Noter : si l'association existait dj => elle est "efface" et "recre".

La mthode public void associerCommande (Commande comm) dans la classe Client ralise le traitement :

  • Si le paramtre comm est null : ne rien faire.
  • Si comm est dj enregistre comme Commande de ce Client : ne rien faire.
  • Si comm est associ un Client : dfaire cette association (removeCommande()).
  • Crer les liens pour associer le Client et comm (setClient() et addCommande()). 
class Client {
        // ...
        public void associerCommande (Commande comm) {
            if (comm != null) {
                if(!this.lesCommandes.contains(comm)) { // Peut tre enlev
                                                                // Efface et recre l'association
                        if (comm.getClient() != null ) {
                                comm.getClient().removeCommande(comm);
                        }
                        this.lesCommandes.add ( comm ); // Ou this.addCommande(comm);
                        comm.setClient ( this );
                }
            }
        }
}

La mthode public void defaireAssoCommande (Commande comm) dans la classe Client :

  • Si le paramtre comm est null : ne rien faire.
  • Si la comm N’est PAS enregistre comme Commande de ce Client : ne rien faire.
  • Dfaire les liens entre le Client et comm (removeCommande() et setClient(null)). 
class Client {
        // ...
        public void defaireAssoCommande (Commande comm) {
            if (comm != null) {
                if(this.lesCommandes.contains(comm)) {
                        this.lesCommandes.remove ( comm ); // Ou this.removeCommande(comm);
                        comm.setClient ( null );
                }
            }
        }
}
