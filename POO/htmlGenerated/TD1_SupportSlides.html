1. Comprendre l’implantation mmoire des donnes

1.1. Principe de stockage des donnes en mmoire : les variables

Une variable permet d’associer

  • un identificateur
  • un type
  • un emplacement mmoire qui contient une valeur

Quel que soit le langage, on parle pour une "variable" (sens dpendant du contexte) de :

  • r-value : rigth value, expression qui a une valeur qui "n’a pas d’adresse", tout ce qui peut se mettre droite d’une affectation (=). Ex : la valeur 10.
  • l-value : left value, expression dont la valeur permet d’accder un emplacement mmoire, tout ce qui peut se mettre gauche d’une affectation (=). Contre exemple : la valeur 10.

Une variable permet, en utilisant l’identificateur :

  • d’accder la valeur → aspect r-value
  • de modifier le contenu de l’emplacement mmoire avec une valeur du type attendu → aspect une l-value

1.4. Emplacement mmoire

Quel que soit le langage, au runtime, un programme dispose de mmoire alloue par le systme d’exploitation :

  • un segment de mmoire pour stocker le code (segment de code),
  • un segment de mmoire pour stocker les donnes (segment de donnes) dcoup en trois parties : i) un sous-segment pour la pile, ii) un sous-segment pour le tas, iii) un sous-segment permanent : donnes globales, donnes statiques des fonctions.

Un programme Java ne s’excute pas directement sur le systme d’exploitation, mais dans la Machine Virtuelle Java (JVM).

Un programme Java ne "dispose" que de deux segments de donnes :

  • un sous-segment pour la pile variables automatiques,
    • il est utilis lors des appels de sous-programmes pour stocker :
      • les paramtres de la signature (en-tte),
      • les variables locales o qu’elles soient dfinies dans le corps du sous-programme,
      • fait partie de ce qu’on appelle le contexte de la fonction, cf. cours systme,
    • sa gestion est "automatique" : allocation pour les variables lors du dbut de la fonction et libration la fin de la fonction,
    • dure de vie des espaces allous : le temps d’excution du sous-programme.
  • un sous-segment pour le tas mmoire gre par le programmeur,
    • il est utilis lors des appels l’oprateur new (tableaux et objets),
    • sa gestion est assure par le programmeur : allocation (new) et libration de la mmoire
      • responsabilit au programmeur de faire les allocations lorsque de besoin,
      • responsabilit au programmeur de librer la mmoire lorsqu’elle n’est plus ncessaire (c’est automatique en Java vous verrez … mais cf. Langage C),
    • dure de vie des espaces allous : entre le new et la libration, indpendamment du sous-programme qui fait le new.
