1.2. Structures de donnes

Une structure de donne dfinit :

  1. un ensemble d’informations organises,
  2. l’organisation de ces informations,
  3. les oprations applicables sur ces informations … et leur rsultat respectif.

Exemples: pile, files, listes, ensembles, map (clef/valeur), arbres, graphes, …

Il faut construire/implmenter ces structures de donnes partir des possibilits de chaque langage.

Une implantation d’une structure de donne repose sur :

  • une reprsentation mmoire choisie,
  • une implmentation algorithmique des oprations de la structure de donne.
  • 2 lments fondamentaux prendre en compte : espace mmoire - temps excution/accs.

Dans les environnements de dveloppement, on trouve des bibliothques (packages java) qui offrent diverses implmentations des structures de donnes.

Par exemple, en java on trouve au moins deux implmentations de la structure de donne liste itrative :

  • ArrayList<E> : liste implante sous forme de tableau,
  • LinkedList<E> : liste implante sous forme de structure doublement chane.

Mais on trouvera aussi :

  • Structure de donne pile :
    • Stack<E> : pile implante sous forme de tableau,
    • et d’autres …
  • Structure de donne file :
    • ArrayDeque<E> : file implante sous forme de tableau.
    • et d’autres …
  • Structure de donne ensembles (collection sans doublons)
    • HashSet<E> : ensemble implant sous forme de paires clef/valeur,
    • TreeSet<E> : ensemble implant sous forme d’arbre balanc.
  • Structure map (association clef/valeur)
    • HashMap<K,V> : map implante sous forme d’une table de hachage,
    • TreeMap<K,V> : map implante sous forme d’arbre balanc.

4. Classe dvelopper : Pile de String

La pile :

  • Structure de type LIFO (Last In, First Out).
  • Dernier lment entr dans la pile sera le premier sorti.

On souhaite dfinir une classe Pile de String dont l’implmentation est un tableau "dynamique" de la taille exacte du nombre d'lments contenus dans la Pile. L’implantation propose ici consiste changer le tableau de stockage chaque modification de la pile (Le type Pile a dj t implment sous forme d’un tableau de taille fixe, 100 lments par exemple).

Diagramme UML complet de la classe Pile Mthodes

Les mthodes :

* Pile() : Constructeur par dfaut qui construit une pile vide (sans lments).

* boolean estVide () : retourne true si la pile est vide, false sinon.

* String sommet () : donne accs au sommet de la pile, l'lment en "haut" de la pile (le dernier empil).

⇒ Prcondition : sommet() valide ssi pilevide() == false. Lve une exception PileException si non respect.

* void empiler (String elt) : permet d’ajouter un lment en "haut" de la pile.

* void depiler () : enlve l'lment en "haut" de la pile.

⇒ Prcondition : depiler() valide ssi pilevide() == false. Lve une exception PileException si non respect.

NB : noter que dans certains cas (livres, sites web, etc.), l’en-tte de depiler() pourrait tre String depiler (), signifiant que depiler() pourrait ET enlever le sommet de la pile ET renvoyer la valeur de ce dernier.

4.2. Codage

crire les mthodes de la classe Pile implmente par un tableau "dynamique" de la taille exacte du nombre d'lments contenus dans la Pile.

En cas d’appel des mthodes hors prconditions, les mthodes lveront une exception de type PileException.

Solution
public class Pile {
        private String[] values;

        public Pile() {
                 this.values = new String[0];
        }
        public boolean estVide () {
                return this.values.length == 0 ;
        }
        public String sommet () throws PileException {
                if (this.estVide()) {
                        throw new PileException();
                }
                return this.values[this.values.length-1];
        }

        public void empiler (String elt) {
                String[] nouvTab;
                int i;
                nouvTab = new String[this.values.length + 1];
                for (i=0; i<this.values.length; i++) {
                        nouvTab[i] = this.values[i];
                }
                nouvTab[nouvTab.length-1] = elt;
                this.values = nouvTab;
        }

        public void depiler () throws PileException {
                if (this.estVide()) {
                        throw new PileException();
                }
                String[] nouvTab;
                int i;
                nouvTab = new String[this.values.length - 1];
                for (i=0; i<this.values.length-1; i++) {
                        nouvTab[i] = this.values[i];
                }
                this.values = nouvTab;
        }

}

class PileException extends Exception {}
