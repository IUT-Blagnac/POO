Attention

Notez que le debugger affiche les objets sous forme "arborescente". De ce fait :

  • Un mme objet peut se "voir" plusieurs endroits.
  • On les reconnat car ils ont le mme id
    • Cas de "Shmi" dont l’id (et la description) se retrouve en descendant de "Anakin"
    • Cas de "Anakin" dont l’id (et la description) se retrouve en conjoint de "Padme"
    • Cas de "Padme" dont l’id (et la description) se retrouve en conjoint de "Anakin"

Notez que les numros d’objets affichs sont diffrents des "id" affichs par le debugger. Ils sont nanmoins quivalents (identifiant unique).

Notez aussi que la valeur null est affiche en 0 par l’instruction System.identityHashCode(…) alors que sinon "null" est affich dans le debugger Eclipse ou sur la console lors d’un affichage normal (System.out.print(…)).
